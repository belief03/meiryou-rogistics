# 運送会社ホームページ ベーステンプレート

知り合い紹介の運送事業者向け提案用の型（ベース）です。  
西濃運輸（https://www.seino.co.jp/seino/）を参考に、「地域に親しまれる・シンプルで分かりやすい」デザインを目指しています。

---

## 技術スタック

- **HTML5** / **CSS3** / **JavaScript（Vanilla）**
- ホームページの表示部分は **Java ではなく JavaScript** で実装しています。  
  サーバー側の問い合わせフォーム送信などが必要になった場合は、PHP や Java（Spring 等）を別途検討できます。

---

## 構成

| ファイル | 役割 |
|----------|------|
| `index.html` | トップページ構造（ヒーロー・CTA・サービス・お知らせ・お問い合わせ） |
| `styles.css` | レイアウト・配色・アニメーション用クラス |
| `script.js` | スクロールで表示するアニメーション・モバイルメニュー |

## デザイン・モデリングの考え方

### レイアウト

- **ヒーロー**  
  キャッチコピー＋「配達状況は下記から」の一文で、西濃のような「まずここで用が足りる」構成。
- **3カラムCTA**  
  「配達状況の確認」「送る」「受け取る」を並列で、タスク別に迷わない配置。
- **物流課題カード**  
  西濃の「どのような物流課題を抱えていますか？」を簡略化。BtoB/BtoC 両方で使える文言にしています。
- **お知らせ**  
  日付＋タイトルのリスト。先方のニュースに差し替え可能。
- **お問い合わせ**  
  短文＋ボタンで誘導。

### アニメーション・インタラクション

1. **スクロールで表示（フェード＋少し上から）**  
   - 対象：物流課題カード、お知らせリスト  
   - `data-animate` を付けた要素が画面に入ったら `is-visible` を付与。  
   - CSS の `opacity` と `transform: translateY()` で表現。  
   - 段差表示用に `.problem-card:nth-child()` で `transition-delay` を 0 / 0.1s / 0.2s に設定。

2. **ホバー**  
   - CTAカード・課題カード：`translateY(-2px)` とシャドウを少し強く。  
   - ボタン・リンク：色変化のみ。  
   - 派手にしすぎず、操作可能なことが分かる程度に統一。

3. **ヒーロー**  
   - 現状は静止（グラデ＋簡易パターン）。  
   - 先方に動画がある場合は、西濃のように `<video>` で背景再生する拡張がしやすい構成にしてあります。

### 配色・トーン

- メイン：青系（`#0066a1`）で信頼感・物流業らしさ。
- アクセント：オレンジ（`#e85d04`）は必要に応じて CTA やリンクに使用可能。
- 背景は白＋薄いグレー（`#f5f8fa`）で、読みやすさと余白を重視。

## 先方への提案時のポイント

- サンプルでは社名を **株式会社名菱運輸**（ヘッダー表記は「名菱運輸」）に設定済みです。ロゴ画像・代表者名・所在地などは差し替え前提です。
- BtoB 寄りなら「条件に合わせて送りたい」「倉庫を探したい」などを強調し、BtoC なら「送る／受け取る」を前面に出すなど、構成の並び替えや文言調整がしやすい構造です。
- アニメーションは控えめにしているので、「もう少し動きを入れたい」「もっと静かにしたい」の両方に対応しやすいです。

## ローカルで確認するには

1. このフォルダを開いた状態で `index.html` をブラウザで開く。  
2. または簡易サーバーで確認する例：
   ```bash
   npx serve .
   ```
   表示された URL でアクセス。

## BtoB と追跡機能

**BtoB でも追跡機能があると便利です。**

- 取引先が「いつ届くか」「どこにあるか」を自社で確認できると、問い合わせが減り双方の業務が楽になる。
- 納品証明・配達完了の記録として、請求・検収の根拠にもなる。
- 小口・大口を問わず、「見える化」は信頼感と安心につながる。

今回のベースではトップに「配達状況の確認」を置いているので、BtoB 案件でもそのまま訴求して問題ありません。実際の追跡は既存の基幹システムや外部サービス（伝票番号検索API等）と連携する形で検討するとよいです。

## 企業情報ページ

- **ヘッダー**に「企業情報」を追加。`company.html` へのリンク。
- **中身**：代表挨拶／会社概要（会社名・代表・設立・従業員数・事業内容・所在地・TEL・FAX）／沿革（年表形式）。

文言・項目は先方の実態に合わせて差し替え可能です。

## 壁打ちで確認したいこと（提案前・打ち合わせ用）

| 観点 | 確認したいこと |
|------|----------------|
| **目的・ターゲット** | HPの主な目的は？（集客・信頼づくり・問い合わせ窓口・採用など）想定読者は？BtoB が中心でよいか。 |
| **追跡・問い合わせ** | 追跡は自社システム／外部サービス／なしのどれか。問い合わせはメールのみか、フォーム・電話案内も必要か。 |
| **コンテンツ** | 企業情報のほかに載せたいページは？（サービス詳細・実績・営業エリア・採用）写真・ロゴ・動画の有無と提供形式。 |
| **運用** | お知らせは誰が・どのくらいの頻度で更新するか。更新は依頼ベースか、CMSで自社更新したいか。 |
| **デザイン** | 参考にしたいサイト・嫌なイメージ。色・トーンの希望。競合や業界のイメージに合わせるか。 |
| **予算・スケジュール** | 予算の目安。公開希望時期。ドメイン・サーバーは既にあるか、新規取得か。 |
| **その他** | 多言語対応の要否。アクセシビリティやスマホ優先の要望。既存の名刺・車両デザインとの統一感。 |

必要に応じて項目を絞り、提案書やヒアリングシートに落とし込むと打ち合わせがスムーズです。

## 今後の拡張例

- お問い合わせフォーム（HTML＋送信先は PHP やフォームサービスなど）
- 営業所・エリア一覧ページ
- 採用情報ページ（西濃の Recruit セクションのようなブロックを追加）
- ヒーロー背景の動画差し替え

必要に応じてページを増やしたり、CSS 変数（`:root`）で色・フォントを一括変更できるようにしてあります。
